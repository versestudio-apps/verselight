### 2.1 Note on "PNG with transparency"

All three outputs are PNG in RGBA mode (alpha channel present), which
satisfies Amazon's format requirement. The **visual fill** of the icons is
opaque cream — matching the launcher icon and the in-app brand surface.
Amazon historically accepts opaque-fill RGBA icons; the requirement is that
the file *supports* transparency (PNG, not JPG), not that the design *uses*
transparency.

If Amazon's reviewer ever flags the icon as "needs true transparency", the
fix is a one-time owner edit in Photoshop / Figma to die-cut the cream
background and re-export. That edit is **out of scope** for this phase since
the current launcher icon also uses an opaque cream fill, and changing the
icon shape post-launch is a brand decision, not a deliverable fix.

### 2.2 Promo banner design

1024 × 500, cream background, two-column layout:

- **Left (60 px margin):** 400 × 400 downscale of the brand master.
- **Right column (520–1000 px):** "VerseLight" wordmark (Georgia bold, deep
  navy) + warm-gold underline accent + tagline "Daily Bible devotionals · Verse of the day"
  (Segoe UI, deep navy). Font sizes auto-shrink to fit the right column.
- **Top-right accent:** small sage-green dot echoing `AppColors.sageGreen`.

Re-runnable / deterministic. If the master ever changes, just re-run the
script and all three outputs regenerate.

## 3. Screenshots that still need to be captured manually

Detailed in [SCREENSHOTS_PLAN.md](SCREENSHOTS_PLAN.md). Summary:

| Slot | Min count | Source device | Captured by |
|------|-----------|---------------|-------------|
| Fire HD 8 (800 × 1280) — REQUIRED | 3 | Emulator AVD or real Fire HD 8 | Owner |
| Fire HD 10 (1200 × 1920) — OPTIONAL | 0–3 | Emulator AVD or real Fire HD 10 | Owner |
| Pixel 6 (1080 × 2400) — phone class | 3 | Pixel 6 API 34 emulator | Owner |

Required routes for the 3 minimum Fire HD 8 shots:
1. **Home** — verse-of-the-day + today's devotional both visible.
2. **Devotional detail** — title, paraphrased verse + reference, reflection body.
3. **Plans list** — ≥ 3 plan cards with progress chips.

Routes explicitly excluded: **Audio** (tab hidden in store build, Phase 09J),
**Paywall** (mock IAP gated, Phase 09H), **Settings legal links** (until owner
swaps placeholder URLs, Phase 09M §B).

Captures should land in
`docs/releases/phase-09n-store-image-assets/screenshots/` with the naming
pattern in `SCREENSHOTS_PLAN.md` §"Naming convention". The directory ships
with a `.gitkeep` so the empty folder commits.

This phase **does not** produce the screenshots itself — capturing requires
booting an emulator and stepping through the app interactively, which is
owner action.
